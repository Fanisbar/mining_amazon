\begin{abstract}
This report presents a comprehensive data mining and analysis study conducted on large-scale Amazon datasets. The project is divided into two complementary parts. Part 1 mainly focuses on Exploratory Data Analysis (EDA) and Sentiment Analysis to explore user reviews, ratings and the overall structure of the datasets. Part 2 involves text pre-processing, clustering of products to group them into subsets, a recommendation system and frequent pattern mining to uncover association rules and purchasing patterns. The work is focused on searching (mining) for useful and interesting information on large scale datasets, the characteristics of which we don't know a priori. The Amazon product categories that were used are the following five. "Appliances", "All\_Beauty", "Musical\_Instruments", "Software" and "Video\_Games".
\end{abstract}

\section{Project Structure and Technical Setup}

This is a project that aims to accomplish an extensive and precise study on large scale Amazon datasets. All of the code is written in Python (v3.10.12) using Jupyter notebooks and is organized into two main parts, corresponding to the two analytical phases of the study. All required dependencies must be installed before running any of the notebooks with the following command at the root of the repository:
\colorbox{backcolour}{\texttt{pip install -r requirements.txt}}
After installation, the notebooks can be executed.

\section{Datasets}
Datasets used in this project are from \href{https://huggingface.co/datasets}{Hugging Face} and concern Amazon products, including reviews and metadata collected in 2023 by McAuley Lab. Both parts use the same 10 csv files: two datasets, reviews and metadata, for each product category. The first few code cells in both notebooks download the datasets locally. One could use the max\_rows variable to limit the datasets to an arbitrary number of rows, to save time and system resources. Full-size datasets are quite large and take up approximately 6.4GB of space.

\begin{itemize}

After experimentation, it was found that no other pre-processing was necessary for this part.
